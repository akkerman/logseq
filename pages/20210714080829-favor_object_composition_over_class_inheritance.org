#+title: Favor object composition over class inheritance
#+options: num:nil toc:nil

Met overerving is de interne werking van een parent class vaak zichtbaar voor subclasses. (white box)

Compositie is een alternatief voor overerving waarbij je meerdere objecte
samenstelt tot nieuwe functionaliteit zonder de interne werking zichtbaar te
hebben. (black box)


* Overerving / Inheritance

*Voordelen*
- directer (eenvoudiger) te gebruiken
- beschikbaar tijdens compilatie
- gemakkelijk in hergebruik; je hoeft bv slechts een deel vd methodes te implementeren om het gedrag aan te passen (vgl [[roam:template method]])

*Nadelen*
- Op runtime kun je de implementatie van de parent niet aanpassen
- Omdat sub-classes details van de parent kunnen zien (eigenlijk breken ze
  encapsulation), zijn ze vaak zo met elkaar verbonden dat een wijziging aan de
  parent een wijziging aan de sub-class behoeft.
